# AR.js 位移追蹤 - 快速開始指南

## 🚀 5 分鐘快速上手

### 步驟 1：準備 Marker（1 分鐘）

1. **下載 Hiro Marker 圖案**
   - 訪問：https://jeromeetienne.github.io/AR.js/data/images/HIRO.jpg
   - 或使用項目中的默認 Marker

2. **打印 Marker**
   - 使用 A4 紙打印
   - 尺寸建議：10cm × 10cm
   - 確保圖案清晰

### 步驟 2：打開測試頁面（30 秒）

1. 在手機瀏覽器打開 `index.html`
2. 點擊頂部 **"📍 位移測試"** 按鈕
3. 允許攝像頭權限

### 步驟 3：開始追蹤（30 秒）

1. 將 Marker 放在桌面上
2. 用手機攝像頭對準 Marker
3. 等待檢測成功（狀態欄顯示 "Marker: 已檢測"）
4. 移動手機，觀察數據變化

### 步驟 4：連接 Unity（2 分鐘）

1. 啟動 Unity 應用程序
2. 確保 WebSocket 連接成功
3. 將 `PositionController` 腳本附加到要控制的物體
4. 開始測試！

---

## 📱 操作說明

### 基本操作

| 操作 | 效果 |
|------|------|
| 手機向上移動 | Unity 物體向上移動 |
| 手機向右移動 | Unity 物體向右移動 |
| 手機向前移動 | Unity 物體向前移動 |
| 手機旋轉 | Unity 物體旋轉 |

### 控制按鈕

- **重置初始位置**：將當前位置設為新的參考點
- **調整縮放**：放大/縮小移動幅度（默認 1.0）
- **設置偏移**：微調 Unity 中的物體位置

---

## ⚠️ 注意事項

1. **需要 HTTPS**：生產環境必須使用 HTTPS，開發環境可用 localhost
2. **光照要求**：確保 Marker 有充足且均勻的光照
3. **距離建議**：保持 20-50cm 的距離
4. **穩定性**：避免快速移動，保持手機穩定

---

## 🔧 常見問題快速解決

| 問題 | 解決方法 |
|------|----------|
| Marker 檢測不到 | 改善光照、調整距離、確保 Marker 清晰 |
| WebSocket 未連接 | 檢查網絡、點擊"重新連接"按鈕 |
| Unity 物體不移動 | 檢查 Unity 控制檯、確認腳本已附加 |
| 位置方向相反 | 調整縮放為負值或修改 Unity 座標映射 |
